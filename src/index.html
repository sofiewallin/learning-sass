Learning CSS preprocessors and Sass

This page is part of a school assignment where I'm learning Sass, but also practising a little bit of CSS animation, JavaScript and Webpack.

The Assigment

The instructions for the assignment was to:

  • Create a simple web template for a fictional website with optional content, design and structure. The site only needs to consist of one HTML file but it's ok to add more. ✓
  • Let the website have at least headings and text, a main menu with five links, a color scheme with a minimum of three colors, two images and be responsive. ✓
  • Generate the website's CSS code from Sass. The development environment of the website should contain all the functionality of the previous assignment and functionality for compiling Sass files into CSS. ✓
  • Let the Sass code at a minimum contain variables, nesting, mixins, selector inheritance, if/else, color functions (darken, lighten at least). ✓
  • Publish the website to a publicly available web host and to GitHub. ✓

Using Webpack

Instead of using Gulp as I did in the previous assignment, I decided to use webpack. Gulp gave me a lot of warnings and errors and didn't seem to be as continuously updated anymore, webpack seems to be what most developers are using today.

I followed a tutorial and the documentation on their website and think I got the hang of it. This is the setup I built:

  • A project folder with Git, npm, webpack and webpack CLI.
  • A remote repository at GitHub.
  • A README.md describing this assigment, how I set up webpack and how to use my repository.
  • A .gitignore that ignores node_modules, dist and .DS_store (because I'm working on a Macbook).
  • Automatically created node_modules/, package.json and package-lock.json.
  • A source folder (src/) with all the code for the website and an index.js file as the webpack entry point.
  • Three config files, one for common configurations webpack.common.js, one for development configurations webpack.dev.js and one for production configurations webpack.prod.js.
  • The website is configured to use dist/ as the websites production folder where all the compiled and ready files go.

I use separate configurations for development mode and production mode. In production all the files are compiled, minimized and ready to be published compared to the development files.

Loaders and plugins

These are the loaders and plugins I've used:

  • css-loader for translating CSS into CommonJS.
  • style-loader for creating style nodes from JS strings.
  • sass-loader for compiling Sass into CSS. This loader depends on Dart Sass so that was used as well.
  • webpack-merge for merging module.exports in seperate config files.
  • HtmlWebpackPlugin and html-loader for moving, minimizing and handling html files.
  • MiniCssExtractPlugin for extracting CSS into seperate file in production.
  • CssMinimizerPlugin for minimizing CSS.
  • DevServer for setting up a development environment and for source mapping.

Setting up Sass

I created a simple setup and structure for my Sass files in src/sass/ based on the information in the article SASS Project Structure for Big Projects by Danny Huang.

I use a main.scss where I import all the partials and set the charset. The partials I use are:

  • _base.scss for resets, variables, mixins and some utility CSS classes.
  • _component.scss for any reusable components.
  • _layout.scss for handling the overall layout of the page.
