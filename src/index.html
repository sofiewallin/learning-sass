Learning CSS preprocessors and Sass

This page is part of a school assignment where I'm learning Sass, but also practising a little bit of webpack.

The Assigment

In this assignment I tried to create a simple grid and layout system with Sass that I can build on later. However, while doing this assignment I've felt that Sass probably has the most perks when working on bigger projects. I found it a bit difficult to see how it was better than pure CSS i some situations because of the limited content of this page.

I feel that what I've built has a lot of potential for improvement, but I think that the result meets all the requirements.

The requirements:

  • Create a simple web template for a fictional website with optional content, design and structure. The website only needs to consist of one HTML file.
  • The website needs to have at least headings and text, a main menu with five links, a color scheme with a minimum of three colors and two images.
  • The website needs to be responsive.
  • The website's CSS code must be generated from Sass.
  • The development environment of the website should contain all the functionality of the previous assignment and functionality for compiling Sass files into CSS.
  • Let the Sass code at a minimum contain variables, nesting, mixins, selector inheritance, if/else and color functions (darken, lighten at least).
  • Publish the website to a publicly available web host and to GitHub.

Using Webpack

Instead of using Gulp as I did in the previous assignment, I decided to use webpack. Gulp gave me a lot of warnings and errors and didn't seem to be as continuously updated anymore, webpack seems to be what most developers are using today.

I followed a tutorial and the documentation on their website and think I got the hang of it. This is the setup I built:

  • A project folder with Git, npm, webpack and webpack CLI.
  • A remote repository at GitHub.
  • A README.md describing this assigment, how I set up webpack and how to use my repository.
  • A .gitignore that ignores node_modules, dist and .DS_store (because I'm working on a Macbook).
  • Automatically created node_modules/, package.json and package-lock.json.
  • A source folder (src/) with all the code for the website and an index.js file as the webpack entry point.
  • Three config files, one for common configurations webpack.common.js, one for development configurations webpack.dev.js and one for production configurations webpack.prod.js.
  • The website is configured to use dist/ as the websites production folder where all the compiled and ready files go.

I use separate configurations for development mode and production mode. In production all the files are compiled, minimized and ready to be published compared to the development files.

Loaders and plugins

These are the loaders and plugins I've used:

  • css-loader for translating CSS into CommonJS.
  • style-loader for creating style nodes from JS strings.
  • sass-loader for compiling Sass into CSS. This loader depends on Dart Sass so that was used as well.
  • webpack-merge for merging module.exports in seperate config files.
  • HtmlWebpackPlugin and html-loader for moving, minimizing and handling html files and assets.
  • MiniCssExtractPlugin for extracting CSS into seperate file in production.
  • CssMinimizerPlugin for minimizing CSS.
  • DevServer for setting up a development environment and for source mapping.

Setting up Sass

I created a simple setup and structure for my Sass files that I can surely improve upon later. I use a main.scss where I import all the partials and set the charset. The partials I use are:

  • _base.scss for resets, variables, mixins and some base style.
  • _typography.scss for text style not related to layout.
  • _layout.scss for handling the general layout of the page.
  • _navigation.scss for handling the navigation.
  • _component.scss for any reusable components, in this case buttons.
